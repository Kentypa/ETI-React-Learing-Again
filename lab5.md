1. Проблема Prop drilling
   Вирішує проблему передачі пропсів через проміжні компоненти, яким ці дані не потрібні. Дані доступні напряму з контексту.

2. Чому Redux/Zustand замість Context API?
   Context API при зміні значення ререндерить усіх споживачів. Redux/Zustand мають механізми селекторів, що оновлюють лише ті компоненти, які залежать від конкретної частини стану.

3. Роль HOC при захисті маршрутів
   Інкапсулює логіку перевірки авторизації. Відокремлює бізнес-логіку доступу від UI-компонентів сторінки.

4. Навіщо replace: true?
   Замінює поточний запис у стеку історії браузера. Після редиректу користувач не зможе повернутися на захищену сторінку (або сторінку логіну після успішного входу) натисканням кнопки "Назад" у браузері.
